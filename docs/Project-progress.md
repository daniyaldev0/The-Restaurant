## Design Consistency

Every new section must feel like it was designed alongside the completed sections.

Maintain consistency in:

* Layout rhythm
* Spacing
* Typography
* Button styles
* Card styles
* Form styling
* Shadows
* Border radius
* Animation timing
* Hover states
* Overall visual hierarchy

Do not introduce new design patterns unless they clearly improve the overall user experience.

---

## Development Rules

* Never modify completed sections unless explicitly instructed.
* Generate only additions to `index.html`, `style.css`, and `script.js`.
* Reuse existing CSS variables, utilities, and helper functions.
* Avoid duplicate code.
* Keep all code production-ready.
* Preserve accessibility and responsive behavior.
* Optimize for performance and maintainability.
