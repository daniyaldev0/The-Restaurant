## Existing Architecture

### HTML

* Single-page semantic HTML5 structure
* Modular section organization
* Accessible markup
* Consistent heading hierarchy

### CSS

* Global design system using CSS variables.
* Shared utility classes.
* Responsive layout.
* Reusable button, card, and form styles.
* Existing animation classes and utilities.

### JavaScript

* Vanilla JavaScript (ES6+).
* Modular architecture.
* Shared helper functions.
* Existing `IntersectionObserver` reveal system.
* Performance-focused implementation.

---

## Design Consistency

Every new section must feel like it was designed alongside the completed sections.

Maintain consistency in:

* Layout rhythm
* Spacing
* Typography
* Button styles
* Card styles
* Form styling
* Shadows
* Border radius
* Animation timing
* Hover states
* Overall visual hierarchy

Do not introduce new design patterns unless they clearly improve the overall user experience.

---

## Development Rules

* Never modify completed sections unless explicitly instructed.
* Generate only additions to `index.html`, `style.css`, and `script.js`.
* Reuse existing CSS variables, utilities, and helper functions.
* Avoid duplicate code.
* Keep all code production-ready.
* Preserve accessibility and responsive behavior.
* Optimize for performance and maintainability.
